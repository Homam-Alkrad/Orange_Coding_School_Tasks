Database Concepts

1. Database Concepts

What is a Database?

A database is a structured collection of data that can be easily accessed, managed, and updated. Databases are used to store information in an organized manner, enabling efficient retrieval and manipulation of data.

What is SQL? What can SQL do?

SQL, or Structured Query Language, is a programming language used to interact with relational databases. SQL can perform tasks like querying data, updating records, creating and modifying database structures, and managing permissions.

What is SQL Server?

SQL Server is a Relational Database Management System (RDBMS) developed by Microsoft. It uses SQL as its primary language for database operations, allowing users to store, retrieve, and manipulate data efficiently.

Microsoft SQL Server Management Studio (SSMS)

SQL Server Management Studio (SSMS) is a software tool developed by Microsoft for managing SQL Server databases. It provides a graphical interface for creating and managing databases, writing and executing SQL queries, and configuring database settings.

SQL Server vs SQL

SQL refers to the language used for interacting with relational databases, whereas SQL Server is a specific RDBMS developed by Microsoft that uses SQL for database operations.

SQL Command Types - What is DDL?

SQL commands are categorized into various types based on their functionality:

Command Type Description SQL Commands
DDL (Data Definition Language) Defines the structure of database objects, such as tables and schemas. CREATE, DROP, ALTER, RENAME
DML (Data Manipulation Language) Manages data within database tables (e.g., INSERT, UPDATE, DELETE). INSERT, UPDATE, DELETE
DCL (Data Control Language) Controls access to the database (e.g., GRANT, REVOKE). GRANT, REVOKE
TCL (Transaction Control Language) Manages transactions within the database (e.g., COMMIT, ROLLBACK). COMMIT, ROLLBACK

Types of DDL

  • CREATE: Creates new database objects, such as tables, views, or indexes.
  • DROP: Deletes existing database objects, permanently removing them from the database.
  • ALTER: Modifies the structure of existing database objects, such as adding or removing columns.
  • RENAME: Renames an existing database object, such as a table or column.

DDL Code Examples

-- Create a new table called 'Employees'
CREATE TABLE Employees (
    EmployeeID INT PRIMARY KEY,
    FirstName VARCHAR(50) NOT NULL,
    LastName VARCHAR(50) NOT NULL,
    DateOfBirth DATE,
    Position VARCHAR(50)
);

-- Alter the 'Employees' table to add a new column
ALTER TABLE Employees
ADD Salary DECIMAL(10, 2);

-- Rename the 'Employees' table to 'Staff'
ALTER TABLE Employees
RENAME TO Staff;

-- Drop the 'Staff' table
DROP TABLE Staff;

Table Operations

Table operations are essential for managing and organizing data effectively. Some common table operations include:

  • CREATE TABLE: Defines a new table in the database with specified columns and data types.
  • DROP TABLE: Deletes an existing table, along with all its data, from the database.
  • RENAME TABLE: Changes the name of an existing table to a new name.
  • ALTER TABLE: Modifies the structure of an existing table, such as adding, deleting, or modifying columns or constraints.

SQL Constraints

SQL constraints are rules applied to columns in a table to ensure the accuracy and reliability of the data. Common SQL constraints include:

  • NOT NULL: Ensures that a column cannot contain NULL values, requiring every row to have a value in that column.
  • UNIQUE: Ensures that all values in a column are unique, preventing duplicate entries.
  • PRIMARY KEY: Uniquely identifies each record in a table. A primary key column must contain unique values and cannot be NULL.
  • FOREIGN KEY: Establishes a link between two tables by creating a relationship between columns, enforcing referential integrity.

SQL Constraints Code Example

-- Create a new table 'College'
CREATE TABLE College (
    CollegeID INT PRIMARY KEY,
    CollegeName VARCHAR(100) NOT NULL
);

-- Create a new table 'Major'
CREATE TABLE Major (
    MajorID INT PRIMARY KEY,
    MajorName VARCHAR(100) NOT NULL,
    CollegeID INT,
    CONSTRAINT fk_College FOREIGN KEY (CollegeID)
    REFERENCES College(CollegeID)
);
