Form Example with action and method

HTML Attribute Descriptions

Required (required): The required attribute makes an input field mandatory. The form will not submit if this field is left empty.

Placeholder (placeholder): The placeholder attribute provides a hint to the user about what to enter in the input field. It appears as light text inside the field until the user starts typing.

Value (value): The value attribute sets the initial value of the input field. This value is pre-filled in the field when the form is loaded.

Readonly (readonly): The readonly attribute makes the input field uneditable. Users can see the value but cannot modify it.

Disabled (disabled): The disabled attribute disables the input field, preventing the user from interacting with it. The value will not be submitted with the form.

Autofocus (autofocus): The autofocus attribute automatically sets focus to the input field when the page loads, allowing the user to start typing immediately.

Minlength (minlength): The minlength attribute specifies the minimum number of characters that the user can enter in the input field.

Maxlength (maxlength): The maxlength attribute specifies the maximum number of characters that the user can enter in the input field.

Form Attributes

Action (action): The action attribute in a form tag specifies the URL where the form data should be sent when the form is submitted.

Name (name): The name attribute is used to identify form controls (like input fields) within the form data that is submitted to the server.

Method (method): The method attribute specifies how the form data should be sent to the server. The two common methods are:

  • GET: Sends the form data appended to the URL in name/value pairs. Useful for form submissions where data retrieval is the main purpose, like search forms.
  • POST: Sends the form data as part of the request body. This method is more secure and is used for actions that modify server data, like submitting forms with sensitive information.
